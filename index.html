Realizing the vision

Iterative & Phased Approach

Iteration : 1

Provision Infrastructure

  • Provision All AWS Services
  • Configuration to drive where & how many

Iteration : 2

Deploy Software Components

  • Containers
  • Java Code
  • Lambdas
  • UI Code

Iteration : 3a

Setup Platform Level Configurations

  • Platform Level Roles
  • Platform Level Users
  • Platform Level Configurations in RDS or DynamoDB

Iteration : 3b

Setup Domain & Subdomain Configurations

  • Domain : Lending , Subdomain : HomeLoan
  • Domain/Subdomain Specific Models
  • Domain/Subdomain Specific Users & Roles
  • Domain/Subdomain Specific Workflows
  • Domain/Subdomain Specific Default Integrations

At end of this iteration "GOLDEN" tenant of this Domain/SubDomain is ready

Iteration : 4

Setup Tenant Specific Configurations

  • Enable client specific integrations

If we are able to automate till this stage our Demo on inception start dream is realized

Iteration : 5

Deploy Incremental Code & Configuration Changes

  • Till now we mastered first time setup
  • Deploy incremental infra changes
  • Deploy incremental model changes
  • Deploy incremental code changes
  • Deploy incremental config changes

Iteration : 6

Promote Code & Configuration Changes across environments

If we have mastered till Iteration 5, this should be just tie up loose ends

The Imperatives

  1. Every iteration achieves near 100% automation
  2. Every iteration once complete is triggered by a pipeline
  3. All artifacts, configurations are version controlled
  4. What we create, we also automate its removal/destruction
